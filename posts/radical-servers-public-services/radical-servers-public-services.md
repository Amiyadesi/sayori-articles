# Riseup 的“激进服务器”列表：公益组织在提供什么，我能做什么

我最近在 Riseup 的安全资源里翻到一页很长的名单

[激进服务器 - riseup.net](https://riseup.net/zh/security/resources/radical-servers)

它收集的不是普通云厂商，也不是免费 VPS 活动

名单里的项目大多由非营利组织、自治技术团体、合作社或志愿者集体维护，为社会运动、草根组织、非商业项目和重视隐私的人提供网络基础设施

有些组织提供邮箱、邮件列表和网站托管，有些维护 VPN、XMPP、Matrix、Etherpad、文件分享、代码托管或社区网络

这页名单让我第一次比较完整地看到，互联网角落中的其他公共服务

## Riseup 为什么把它们叫作激进服务器

Riseup 对这份名单的说明很直接

它收录反资本主义、反等级、自治、女性主义或其他激进技术项目

其中一些团体为草根行动者提供免费服务或互助服务，另一些更关注隐私、自由软件和去中心化

共同点不是技术栈，而是不把赚钱当作第一目标

Riseup 也明确写了，这份列表不等于官方背书

每个组织所在地区、法律环境、日志政策、注册方式和服务状态都不同，使用前仍要自己阅读条款与隐私说明

## 这份名单里有什么

页面按 World Wide、South America、Europe、North America 和 In memoriam 分类

世界范围的项目里有 hackerspaces.org、HackThisSite、Indymedia 和 Take Back The Tech

它们分别偏向社区空间、网络安全学习、独立媒体和反对针对女性的数字暴力

南美部分列出 Codigosur 与 Colnodo

这些组织会为社会运动和非营利团体提供托管、邮箱、邮件列表、流媒体、协作编辑与技术支持

欧洲部分数量最多

这里能看到 Disroot、Framasoft、GreenNet、NoLog.cz、systemli、Syster Server、Koumbit 等不同方向的组织

它们提供的服务跨度很大

- 邮箱与邮件列表
- 网站、博客、Wiki 和论坛托管
- XMPP、Matrix、Mumble 和视频会议
- Etherpad、CryptPad 与其他协作工具
- Git、项目管理与文件分享
- VPN、DNS、社区网络和虚拟服务器
- 面向社会运动、女性主义团体或非商业项目的技术支持

北美部分包括 Riseup 自己，也收录了 Koumbit 和 Anarchy Planet 等项目

页面最后还保留一组 In memoriam

这些项目已经停止或只剩维护模式，却没有被简单从历史里擦掉

这一段很重要，因为公共服务会结束，维护者退出、资金变化和法律风险都可能让一个曾经可靠的入口消失，这也就是公益所必须承担的风险啊

## 几个值得单独认识的项目

### Disroot

[Disroot](https://disroot.org/) 是位于阿姆斯特丹的非营利基金会，由志愿者维护并依赖社区支持

它把邮箱、文件、聊天、协作文档、加密粘贴板和联邦网络工具放在一个开放服务集合里

我之前单独写过 [[disroot-open-services-note|Disroot：一个很像互联网老理想的开源服务集合]]

### Framasoft

[Framasoft](https://framasoft.org/) 来自法国，长期推广自由软件、开放教育和去中心化互联网

它做过大量替代中心化平台的公共工具，也持续维护软件、文档和教育资源

### systemli 与 NoLog.cz

[systemli](https://www.systemli.org/) 为政治活动者和社会运动提供注重隐私的通信服务

[NoLog.cz](https://nolog.cz/) 是捷克的活动者技术集体，提供 Matrix、文件分享、协作文档、视频和其他自由软件服务

这类组织通常不会承诺面向所有人的无限资源，而会把支持对象、申请方式和使用边界写得很明确

### GreenNet 与 Koumbit

[GreenNet](https://www.greennet.org.uk/) 为和平、环境和人权团体提供互联网服务、网站与托管支持

[Koumbit](https://www.koumbit.org/) 是蒙特利尔的非营利组织，为社区团体和社会正义项目提供自由软件技术服务，也有团结托管计划

它们提醒我，公益技术服务不一定全部免费

合理收费、交叉补贴、捐赠和志愿劳动都可能是维持服务的方法

## 使用这份名单前要先确认三件事

我在 2026 年 7 月 19 日实际访问了这份列表，目录仍包含 World Wide、South America、Europe、North America 和 In memoriam

这只能证明 Riseup 的目录页面当前可访问，不能替名单里的每一个组织证明服务仍然开放

第一，检查项目现在是否仍在运行

这份页面保留了历史条目，部分服务说明和配额可能已经过时

第二，检查谁适合申请

有些服务只面向特定地区、社会运动、非营利组织或受邀请用户，不是看到免费就能注册

第三，检查法律与数据边界

隐私友好不等于绝对匿名，也不等于可以忽略备份、账号恢复和当地法律

重要资料仍要保留自己的加密备份，不把任何单一组织当成永远不会关闭的基础设施

## 我能从这份名单学到什么

我没有把 sayori.org 注册成公益组织，也没有这些团体的资金、成员和长期治理经验

因此我不会把自己的小服务和 Riseup、Disroot 放在同一个规模上比较

我能学习的是它们如何说明服务对象、公开边界、依赖捐赠、限制滥用，并承认服务可能停止

比起继续堆很多入口，我更想保留少量真正能维护的服务

## sayori.org 目前开放的服务

完整状态放在 [sayori.org 公共服务说明](https://sayori.org/zh/services/)

### GeoScore 网页审计

[GeoScore](https://geo.sayori.org/) 是无需注册的 SEO 与 GEO 事实审计工具

它面向公开网页，整理站点画像、抽样页面、检查证据、优先修复项和 Markdown 报告

它不会自动修改目标网站，也不会把预测结果包装成真实 AI 引用率

### GeoScore 公开审计 API

我目前还能提供一个公开 API 服务

入口是 [geo-api.sayori.org](https://geo-api.sayori.org/)

它提供 GeoScore 的稳定公开接口，可以运行公开域名审计、读取部署元数据、获取 PageSpeed 证据，并在已有审计上生成 Evidence Map 或 FixPack

API 的请求结构、响应和错误码记录在 [OpenAPI 文档](https://geo-api.sayori.org/openapi.json) 中，中文使用说明在 [GeoScore Docs](https://geo.sayori.org/docs)

最小的站点审计请求如下

```bash
curl "https://geo-api.sayori.org/api/audit/example.com"
```

这个接口返回 Server-Sent Events 流，站点模式最多抽取五个 HTML 页面

运行限制以 [`/api/meta`](https://geo-api.sayori.org/api/meta) 的线上结果为准

它只接受公网 hostname，不支持 localhost、内网地址、IP literal 或任意代理请求

公开 API 也按尽力而为原则运行，遇到额度耗尽、滥用或上游故障时会限流或暂停

### Amiya 的白板

[Amiya 的白板](https://board.sayori.org/) 是一个无需注册的轻量浏览器白板

它适合临时画图、整理草图、讲清楚一个页面结构，或给别人发一张可以直接打开的协作纸面，临时画一些流程图或者其他的展示自己思路的东西还是很好用的

白板仍按尽力而为原则维护，不承诺长期保存重要内容，使用者应该自行导出需要保留的结果

### 博客、RSS 与反馈

[博客](https://blog.sayori.org/) 和 [RSS](https://blog.sayori.org/rss.xml) 用来公开文章、教程和项目记录

留言板与邮件负责接收服务问题和改进建议

## Search Gateway 的边界

[Search Gateway](https://github.com/Amiyadesi/search-gateway) 仍然开源，可以自行部署，也提供兼容 API 与 MCP 接入方式

但 `search.sayori.org` 是站长内部实例，需要鉴权，不提供匿名调用、共享 Token 或公共搜索额度

公开 GeoScore API 与内部 Search Gateway 必须分开说明

前者是可以调用的公共服务，后者是可自部署的软件和私有基础设施

## 下一步

我想继续整理这份激进服务器名单，优先确认仍在运行、中文用户能理解、服务边界写得清楚的组织

对自己的站点，我会先把 GeoScore 网页版、公开审计 API、博客和 RSS 维护稳定

如果以后增加网站体检互助、静态站上线协助或少量搜索证据额度，也会先邀请、限额和人工处理

公共服务的价值不在入口数量

更重要的是使用者知道它能做什么，维护者知道自己什么时候应该说不
