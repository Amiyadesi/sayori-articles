# 我做了一个小型论坛列表仓库

我之前一直想整理一张互联网社区地图

这里说的不是 X、Instagram、贴吧、小红书、小黑盒这种明面上的大平台，而是那些你不主动搜索、不混相关圈子，基本不会自然刷到的论坛和社区

这件事现在已经从一篇文章，变成了一个可以长期维护的独立仓库和在线页面：

- GitHub 仓库：[https://github.com/Amiyadesi/small-forums-list](https://github.com/Amiyadesi/small-forums-list)
- 在线页面：[https://forums.cc.cd/](https://forums.cc.cd/)

现在这个站更接近 wiki 或 docs 的形态：左边是分类和社区目录，中间是具体条目，每个社区都有自己的页面，页面底部也接了 Giscus 评论区

目前首版收了 49 个社区。后面如果有人补充、纠错或者提交 PR，这个数字还会继续变

## 这个仓库收什么

这里的“小型”不是严格按用户数算，而是按接触路径算

如果一个社区不主动搜基本接触不到，或者只有混主机圈、ACG 圈、安全圈、硬件圈、站长圈、资源圈的人才会经常看到，那它就可能适合收录

目前大概分成这些类：

- 技术 / AI / 开发者
- 主机 / VPS / 站长 / 独立站
- 搜索 / 资源 / 知识工具
- 安全 / 逆向 / 系统 / 软件
- ACG / 游戏 / 小圈子
- 硬件 / 电子 / Homelab
- 成人 / 资源 / 灰区
- 海外老论坛 / 泛兴趣

虽然AI分的感觉有些怪，但是没办法就先这样好了

## 现在的页面长什么样

之前它更像一份 README 长列表，现在已经拆成了站点。

首页会保留分类速览，适合快速看有哪些社区；左侧目录适合按分类跳转；每个社区也有单独页面，方便别人直接指出某个社区写错了、入口变了、注册规则变了

我一开始想做的是“互联网社区地图”，但越写越发现，地图最难的不是画出来，而是持续维护

论坛入口会变，注册方式会变，社区氛围也会变。主机圈会因为商家、线路和活动不断出现新信息，ACG 和资源社区也经常因为域名、权限、版规变化而过时

所以现在的思路是：仓库负责做目录和事实记录，博客负责写个人体验

如果我真的长期混过某个社区，就单独写文章。如果只是公开资料整理，就留在列表里

## 每个社区会写什么

仓库里每个条目会尽量写这些信息：

- 入口
- 定位和社区氛围
- 注册方式或门槛
- 里面主要有什么
- 适合看什么
- 注意事项
- 公开来源
- 最后核验日期

我长期进去体验过的社区，会写得更像真实感受。

比如 Linux.do 和 NodeLoc，我会参考自己之前那篇[[internet-community-1|互联网社区记录1]]里的体验去写：Linux.do 的规则、新手任务、公益站、福利羊毛、开发调优、文档共建、LDC；NodeLoc 的自由氛围、子论坛、电池积分、AFF 和站长活跃度

没有长期使用过的社区，就只按公开页面和圈内说法整理，不伪装成亲身体验

## 为什么欢迎 PR

这类列表靠一个人肯定会错

有些社区外人只能看到门口，看不到真实氛围。有些论坛注册规则会阶段性开放，有些域名会漂，有些旧称和新称很容易混在一起

所以这个仓库最需要的不是更漂亮的介绍，而是社区真实成员来修正

如果你是某个社区的老用户，或者只是发现某个条目写错了，可以直接提 PR。最有价值的补充通常是这些：

- 注册现在是开放、邀请制，还是阶段性开放？
- 新人进去应该先看哪个版规或公告？
- 社区真实氛围偏技术、偏灌水、偏交易，还是偏资源？
- 里面最有价值的板块是什么？
- 入口是否变了，旧域名是否已经不稳定？
- 某些描述是否太像 AI 总结，需要改成真实用户口吻？

如果不想提 PR，也可以直接在对应社区页面下面留言。站点现在接了 Giscus，评论会落到 GitHub Discussions 里

## 如何协助整理

新增或修改社区时，优先改仓库里的结构化数据：

```bash
data/communities.json
```

改完之后运行：

```bash
node scripts/generate-readme.mjs
node scripts/build-site.mjs
```

前者会同步 README，后者会生成在线页面

如果只是改分类说明，可以看：

```bash
docs/categories.md
```

如果是补充单个社区，最好同时给公开来源。来源可以是官网、注册页、站内公告、关于页面、版规页面，或者其它能公开访问的说明

## 目前的整理原则

这个项目目前主要由 AI 根据公开资料、站点页面和少量个人体验整理。只有一部分社区是我真实长期进入体验过的

所以我希望后面每个社区都能慢慢被真实用户修过一遍

成人、资源、破解、逆向、安全、刷机、交易这些社区也会收，但会按大类写清楚：里面大概有什么、适合观察什么、有什么坑。具体是否进入、是否参与、怎么判断风险，都需要读者自己负责

我现在更想把它做成一份能长期改的公共目录，而不是一篇写完就过时的博客文章

如果你发现有我没写到的论坛，或者某个条目写得很外行，欢迎直接去仓库提 PR：
